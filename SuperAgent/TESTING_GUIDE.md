### No Gemini API Logs

If you don't see detailed Gemini API logs, check:

1. **Environment variable**: Ensure `GOOGLE_API_KEY` is set in `.env`
2. **Log level**: Set `LOG_LEVEL=DEBUG` in `.env`
3. **Python logging**: The server should show `DEBUG` level logs

## 🎯 Expected Behavior

### Natural Two-Step Flow

The system implements a **natural conversational flow**:

1. User provides company info → **DiscoveryAgent** responds
2. Agent asks if user wants serviceability check
3. User says "yes" → **SuperAgent** routes to **ServiceabilityAgent**
4. Each user message is one agent turn (no automatic recursive routing)

### Agent Hierarchy

```
SuperAgent (orchestrator)
├── GreetingAgent (greetings, phone scripts)
├── DiscoveryAgent (company registration, BANT qualification)
├── ServiceabilityAgent (address validation, coverage check)
├── ProductAgent (product catalog, specs)
├── PaymentAgent (credit checks, payment processing)
├── ServiceFulfillmentAgent (installation scheduling)
└── FAQAgent (support questions, policies)
```
